# 基本操作
## 介绍

上一篇我们下载下来的example-app是开发员的workspace或App。App是建立在Core之上，也可以说成App是子类(child),而Core是父类(parent),两者是继承(inheritance)关系。

Core的部分，开发员只能调用而不能改动。打开example-app/composer.json。
```sh
{
    "name": "laravel/laravel",
    "type": "project",
    "description": "The Laravel Framework.",
    "keywords": ["framework", "laravel"],
    "license": "MIT",
    "require": {
        "php": "^7.3|^8.0",
        "fruitcake/laravel-cors": "^2.0",
        "guzzlehttp/guzzle": "^7.0.1",
        "laravel/framework": "^8.75",
        "laravel/sanctum": "^2.11",
        "laravel/tinker": "^2.5"
    },
    ...
```
这里通过composer下载的Core包（"laravel/framework": "^8.75"）将被存放在vendor/文件夹。
