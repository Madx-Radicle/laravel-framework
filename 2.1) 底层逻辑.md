# 底层逻辑
首先public/index.php就是每个用户请求的入口处。
```sh
<?php
# [Illuminate\Contracts](https://github.com/laravel/framework/tree/8.x/src/Illuminate/Contracts)包里包含了全部laravel framework的interface
# interface的用意就是可以让开发人员可以自己客制化和扩展。
use Illuminate\Contracts\Http\Kernel;

# https://github.com/laravel/framework/blob/8.x/src/Illuminate/Http/Request.php包
use Illuminate\Http\Request;

define('LARAVEL_START', microtime(true));

if (file_exists($maintenance = __DIR__.'/../storage/framework/maintenance.php')) {
    require $maintenance;
}

require __DIR__.'/../vendor/autoload.php';

$app = require_once __DIR__.'/../bootstrap/app.php';

$kernel = $app->make(Kernel::class);

$response = $kernel->handle(
    $request = Request::capture()
)->send();

$kernel->terminate($request, $response);

```
